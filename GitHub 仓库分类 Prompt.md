# GitHub 仓库分类 Prompt

将以下 prompt 复制到 Cursor 中，让 AI 根据你的仓库列表生成分类文档。

---

## Prompt 模板

```
请根据以下 GitHub 仓库列表，按照合理的逻辑进行分类。

仓库所有者: [你的用户名]
仓库列表:
[仓库列表]

请按照以下要求生成分类文档：

1. 使用 Markdown 格式
2. 每个分类使用二级标题（##），格式为：`## 分组名 <!-- 代号 -->`
   - 分组名：中文描述（如"Go 学习"）
   - 代号：使用军事代号风格（如"597.9高地"、"537.7高地"、"54号高地"、"140高地"等），用于命令行快速输入。使用时可以不带"高地"后缀（如"597.9"、"54号"）
3. 每个分类下使用无序列表列出仓库名称
4. 可以添加分类说明（可选）
5. 分类逻辑可以是：按编程语言、按学习阶段、按项目类型、按用途等

**代号命名建议**：参考 `高地编号参考.md`，使用历史上著名的高地编号，如：
- 上甘岭战役：597.9高地、537.7高地
- 老山战役：54号高地、50号高地、56号高地
- 二战：140高地、A高地、B高地等

输出格式示例：
## Go 学习 <!-- 597.9高地 -->
- Practice-Go-by-Example-
- GoLearn
- goWebActualCombat

## Java 学习 <!-- 537.7高地 -->
- JavaGuide
- toBeBetterJavaer

## 工具 <!-- 54号高地 -->
- get_jobs
- ocr-mcp-service
```

---

## 使用方法

**在 Cursor 对话框中执行**

1. 在 Cursor 对话框中执行：`@PROMPT.md 执行当前prompt`
2. AI 会获取你的仓库列表并生成分类文档，展示在对话框中
3. 检查分类是否合理，如需调整告诉 AI 你的修改意见
4. 确认满意后，告诉 AI "保存" 或 "持久化"，AI 会保存为 `REPO-GROUPS.md`

**同步分组**

完成分类后，使用同步脚本批量同步分组：

```bash
# 批量同步单个分组（使用代号，推荐不带"高地"）
bash sync-groups.sh 597.9

# 批量同步多个分组
bash sync-groups.sh 597.9 537.7 54号

# 列出所有可用分组
bash sync-groups.sh --list
```

💡 **提示**：每个分组会自动创建对应的文件夹（如 `597.9/`、`54号/`），可以直接打开文件夹作为 workspace。
